What is HTML, CSS, and JS?

By Vincent

What is HTML, CSS, and JS? They are all different types of coding languages but they are generally all linked together to form a webpage. HTML is used mainly for creating the attributes and basic layout of the website. CSS is best used for the design and looks of the website. And JavaScript is used for the functioning, and the altering of the website while in use with variables and functions. All three of the languages have a different look and way of code. However, on this website, created with HTML, CSS, and JS, you will be taught how to use HTML, CSS, and basic JavaScript.

HTML

HTML stands for HyperText Markup Language, which was founded by Tim Berners-Lee in 1993. It has then been improvised to HTML 4.01 in 1999, which is most commonly used today. It is a format of texts using tags to make the properties of a webpage e.g. textboxes, labels, etc.

JavaScript

JS, or JavaScript, is a high level coding language similar to python founded by Brendan Eich, with variables and loops. It is a high level language so I won't be going over every single function because then you'd be reading about 1000 more lines.

Defining variables

Defining variables is one of the most important yet basic parts of the JavaScript Program. Without variables, you cannot make a website consisting of changing numbers at all because you don't have variables to alter and so therefore cannot change because there is nothing to change. Therefore, you should always define variables first.


There are two types of variables, namely var(single variable), const(unchangeable variable, which will give an error if you try and change it or redeclare it), and let(which also throws and error if redeclared). All three, except for var, have block scope which means when you declare them inside braces{}, they can not be used outside or there will be an error.

JS Datatypes

JS Datatypes are basically types of variables well, variables can be. This for example is string of texts, numbers, boolean, etc. Generally different Datatypes can not be compared to each other, otherwise an error will be thrown. For example, you can't use. “If 0 == “0”:” as an if loop because strings can’t simply be compared to numbers. Here are some of the Datatypes.


Text String

Text strings can be pretty much anything, as long as its wrapped in “”. However, there are a few limitations. If you would like to put quotation marks in, you must put a back slash in front of the character. All inputs from text boxes always start as a text string.


Numbers

Numbers are well, numbers! They are always standard Arabic Numerals and are never wrapped in anything. They are used for adding, dividing, subtracting, multiplying, finding integer division and even remainders! This variable is the key to making games involving scores.

Boolean

Booleans are pretty simple to use. They return two values, true and false. Booleans are used generally for "while" loops(a loop that recurs again and again as long as a condition is met).

Undefined

The undefined variable speaks for it self. It means the variable is undefined.

Null

Null is an unusual type of datatype. It means "Nothing" even though a variable defined as null is an "object."

BigInt

BigInt is a datatype that represents very big integers(why else would it be called that?). With BigInts, you can safely store and use large integers even beyond the safe integer limit.

Symbol

Symbol is a datatype of JavaScript, along with string, number, boolean, null and undefined. Its value is stored for internal use.

CSS

CSS stands for Cascading Style Sheets. It is a form of code with mainly braces{}, was founded by Hakon Wium Lie on 10 October 1994 and is used for the design of the HTML website.
The way this works is by using "id=?", or "class=?" property in html elements.

The way you refer to a class or id is in this way...

For example, if your element's id was called "element", this is how you would make it red...

#element

{

background-color:red;

}

It's almost the same with if it was a class...

.element

{

background-color:red;

}

Also, if you want to refer to all the elements that are h1, this is the code:

h1

{

background-color:red;

}

It is very important to keep the semicolon(;) at the end because if you have more properties you want to assign, it won't work properly. It would just keep the first property and ignore the rest.

  • text-underline-position | Specifies the position of the underline which is set using the text-decoration property
  • top | Specifies the top position of a positioned element
  • transform | Applies a 2D or 3D transformation to an element
  • transform-origin | Allows you to change the position on transformed elements
  • transform-style | Specifies how nested elements are rendered in 3D space
  • transition A shorthand property for all the transition-* properties
  • transition-delay Specifies when the transition effect will start
  • transition-duration Specifies how many seconds or milliseconds a transition effect takes to complete
  • transition-property Specifies the name of the CSS property the transition effect is for
  • transition-timing-function Specifies the speed curve of the transition effect

    U

  • unicode-bidi | Used together with the direction property to set or return whether the text should be overridden to support multiple languages in the same document
  • user-select | Specifies whether the text of an element can be selected

    V

  • vertical-align | Sets the vertical alignment of an element(top, bottom, justify etc.)
  • visibility | Identifies if an element is visible or not

    W

  • white-space | Specifies how white-space inside an element is used or proportioned
  • widows | Sets the minimum number of lines that are be left at the top of a page or column
  • width | Decides the width of an element
  • word-break | Specifies how words should break when reaching the end of a line
  • word-spacing | Increases or decreases the space between words in a text
  • word-wrap | Allows long, unbreakable words to be broken and wrap to the next line
  • writing-mode | Specifies whether lines of text are laid out horizontally or vertically

    Z

  • z-index | Sets the stack order of a positioned element(for example, if you put z-index:99999px; you are putting it at the very top.)
